Design a TV application UI for an AI ambient world system.

This is NOT a mobile app and NOT a content browsing platform.

It is a “living world engine” for a living room TV.

Users interact using a remote control from 3–4 meters away.

====================================
1. CORE STRUCTURE
====================================

The screen must be dominated by ONE SINGLE HERO WORLD.

Hero area occupies 100% of the screen background.

Examples of world:
- rainy night cafe window
- mountain cabin fireplace
- midnight city street
- space window floating view

The world is always full screen and always visible.

There must NOT be multiple main content areas.

No grid layout.
No multi-card layout.
No list browsing as default state.

====================================
2. UI LAYERS (VERY IMPORTANT)
====================================

Layer 1 (Base Layer):
- Full screen cinematic world scene
- atmospheric lighting
- slow motion environment (rain, fog, fire, snow)

Layer 2 (Minimal Overlay UI):
UI must be extremely subtle and semi-transparent.

Default state:
UI is hidden

When user interacts:
UI fades in smoothly

When idle:
UI fades out automatically

====================================
3. TOP NAVIGATION (WORLD SWITCH)
====================================

Top center (overlay, minimal):

World names:
- Tokyo Midnight
- Rain Cafe
- Mountain Cabin
- Space Window

Style:
- small text
- centered alignment
- low opacity glass effect
- cinematic spacing

This is NOT a persistent menu.
It is an overlay switcher.

====================================
4. MAIN INTERACTION MODEL
====================================

Default screen shows ONLY the current world.

When user presses remote control:

- Top world switch appears
- Bottom control bar appears

No permanent UI elements.

====================================
5. BOTTOM CONTROL BAR (MINIMAL)
====================================

Bottom overlay (only visible on interaction):

4 actions only:
- Switch World
- Sound
- Favorite
- Settings

Design:
- glass blur background
- low opacity
- minimal icons
- no text overload

====================================
6. HERO WORLD RULES
====================================

The world must feel like:

- cinematic film frame
- living environment
- breathing atmosphere

It must include:

- subtle motion (rain, fog, light flicker)
- depth and lighting layers
- soft environmental dynamics

No UI should block the center of the scene.

====================================
7. MOTION DESIGN
====================================

All motion must be:

- slow
- ambient
- non-distracting
- continuous

Examples:
- rain falling slowly
- steam rising
- snow drifting
- light flicker
- distant moving lights

No fast animations.
No game-like effects.

====================================
8. VISUAL STYLE
====================================

Style direction:
- cinematic
- Apple TV screensaver aesthetic
- Netflix hero-style framing
- premium minimal UI
- dark mode first

Color:
- deep blacks
- soft blues
- warm amber highlights

====================================
9. CRITICAL DESIGN PRINCIPLE
====================================

The entire experience must feel like:

"The world is the product, not the interface."

UI should feel secondary and almost invisible.

====================================
10. IMPORTANT CORRECTION FROM ORIGINAL DESIGN
====================================

Do NOT use:
- multi-card homepage
- content grid layout
- browsing interface
- persistent top navigation bar

Instead:
- single immersive world
- overlay-based switching system
- focus-driven UI only

====================================
FINAL GOAL
====================================

User experience should be:

"I turned on the TV and entered another world."
